# Data Shopping List

What to collect for Week 2-3 RAG and training data.
Priority order — start from the top.

## Priority 1: JLCPCB Parts Database

**What:** Full JLCPCB component catalog with basic/extended classification
**Where:** https://jlcpcb.com/parts or the mixelpixx KiCAD-MCP-Server which has JLCPCB search built in
**Format needed:** JSON/CSV with columns: part_number, value, package, classification (basic/extended), stock, category
**Why first:** This replaces our static lookup table in component_selector.py. Every design decision flows through "is this a basic part?"
**Effort:** Clone mixelpixx MCP server and extract their parts DB integration, or scrape the JLCPCB parts page

## Priority 2: TI WEBENCH Reference Designs

**What:** Power supply designs generated by TI's WEBENCH tool for common specs
**Where:** https://www.ti.com/design-resources/design-tools-simulation/webench-power-designer.html
**Format needed:** For each design: input spec (Vin, Vout, Iout), component list with values, and layout guidelines
**Why:** These are "spec in -> design out" pairs — exactly what we need to teach the LLM layer what components to pick for a given spec
**Effort:** Generate 20-50 designs across buck/boost/LDO topologies at common voltage/current combos. Can be automated with the WEBENCH API.

## Priority 3: EEVblog PCB Review Forum

**What:** Threads where someone posts a PCB layout and experienced engineers critique it
**Where:** https://www.eevblog.com/forum/projects/ (look for "[PCB Review]" tagged threads)
**Format needed:** Image + critique text pairs, classified by issue type (thermal, SI, DFM, placement, routing)
**Why:** Supervised training data for layout quality assessment
**Effort:** Scrape 50-100 review threads. SMF forum software, easy to scrape. LLM-assisted labeling for classification.

## Priority 4: r/PrintedCircuitBoard Reviews

**What:** Reddit posts with PCB reviews and expert feedback
**Where:** r/PrintedCircuitBoard (filter for "Review" flair or posts with images)
**Format needed:** Same as EEVblog — image + critique pairs
**Why:** More layout review data, different community perspective
**Effort:** PRAW library or Pushshift archives. Maybe 100-200 relevant posts.

## Priority 5: Manufacturer Application Notes

**What:** Circuit design guidelines from TI, Analog Devices, Microchip, ON Semi
**Where:** Each manufacturer's website, typically under "Application Notes" or "Design Resources"
**Format needed:** PDF -> extracted design rules as structured JSON
**Why:** These are authoritative design rules (trace widths, cap placement, thermal management)
**Suggested starting set:**
- TI SLVA477 (Understanding Buck Power Stages)
- TI SNVA057 (Inductor and Capacitor Selection for Buck)
- ADI AN-1148 (Layout for Linear Regulators)
- TI SLYT614 (PCB Layout for Switching Regulators)
**Effort:** Download PDFs, extract rules with LLM, add to constraints/ as YAML

## Priority 6: KiCad Open Hardware Reference Designs

**What:** Well-documented open source KiCad projects
**Where:** Adafruit (github.com/adafruit), SparkFun (github.com/sparkfun), OSHW projects
**Format needed:** .kicad_sch + .kicad_pcb files, parsed for component values and layout patterns
**Why:** Real-world validated designs showing how experienced engineers lay out common circuits
**Effort:** Clone repos, parse KiCad files

## Data I Don't Need Yet

- YouTube transcripts (too noisy, low signal-to-noise)
- Raw datasheets (too large, better to extract rules first)
- DeepPCB defect dataset (useful for DFM checker, not for generative design)
- Flux AI distillation (save for later, ToS concerns)
